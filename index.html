  • Superäly: Tiet, Vaarat ja Strategiat

    31.7.2015

    Viime aikoina olen lukenut, tai pikemmin kuunnellut, kirjoja ja sain eilen loppuun Nick Bostromin kirjan Superintelligence: Paths, Dangers, Strategies.

    Kirjan aihe on mielenkiintoinen, kuinka tekoäly tai jokin muu ihmistä-tehokkaampi-superäly muuttaa maailmaa ja kuinka siihen tulisi varautua, mutta kirja esittää paljon terminologiaa jota käytetään harvakseen ja lyhenteitä joita käytetään koko ajan, mutta molempien selitykselle annetaan sama määrä aikaa jolloin helposti menee ohitse tärkeä lyhenne jota kirjassa käytetään usean kappaleen ajan.

    Kun minulle suositeltiin tätä kirjaa, sain kuulla että kirjan alku on tylsä, mutta kahdeksannen kappaleen jälkeen se muuttuu mielenkiintoiseksi, joten lähdin kirjaan pienellä varauksella. Alku ei kuitenkaan ollu loputtoman tylsä tai sekava, asiat tuotiin esille ehkä hieman liian hitaasti ja kaikkea pohdittiin erittäin pitkään. Mielestäni kirja oli keskivaiheilla parhaillaan, kun se esitti erilaisia superäly vaihtoehtoja ja mahdollisia vaaroja kontrolli ongelmien kanssa, mutta nämä ongelmat tuntuivat vain jatkuvan ja jatkuvan. Loppupuolella kirjaa tuntui että samoja ajatuksia pyöritettiin edestakaisin ilman sen suurempia johtopäätöksiä suuntaan tai toiseen. En tiedä johtuiko kirjan kiertely ja kaartelu siitä että kirjoittaja on filosofian tohtori ja täten kaikki ajatukset on pakko pureskella luuytimeen asti, vai oliko kyseessä vain vähän huono kirja.

    Kirja on hieman liian tekninen ja etenee liian hitaasti ja se tuntuu ajoittain jaarittelevalta. Jos tekoäly kiinnostaa aiheena, voi kirja olla hyvä, mutta hieman raskas luku. Itselle olisi hyvin riittänyt puolta lyhyempikin epos.

  • PVCC #13-20: Mega postaus

    29.7.2015

    Etkö tiedä mikä on PVCC? Klikkaa tästä.

    Koska tehtävät 13. - 20. ovat lähinnä Googlen käyttöä pistän ne kaikki tähän yhdeksi postauksesta. Käyn tehtävät lävitse hyvin pintapuolisesti, jos jotain kiinnostaa tietää tarkemmin niin Wikipedia on aika hyvä lähde näille.

    13. Selaimella hyökkäys

    Mitä seuraavista haavoittuvuuksia voidaan hyödyntää käyttämällä vain normaalia web selainta (kuten Mozilla Firefoxia tai Google Chromea)?

    1. Directory traversal attack
    2. LDAP injection
    3. Command injection
    4. Poison Null Byte
    5. SQL injection
    6. XSS

    Tässä tehtävässä oletan että “normaaliksi” selaimeksi lasketaan myös helposti saatavat selain lisäosat, ne eivät toki ole pakollisia kaikkiin hyökkäyksiin, mutta helpottavat niitä huomattavasti. Kaikki listatut hyökkäykset ovat käyttäjän syötteen sanitoimatta jättämisestä johtuvia ongelmia, eli kaikkiin vastaus on kyllä, kaikkia näitä haavoittuvuuksia voi hyväksikäyttää vain selaimella ilman erillisiä työkaluja.

    14. 32 bittiä vs. 64 bittiä

    Mitkä seuraavista ovat tyypillisiä x86 järjestelmälle, mutta ei x86_64 järjestelmälle?

    1. Puskurin ylivuoto haavoittuvuus
    2. DDOS hyökkäyksiä on usein vaikea estää
    3. Roikkuvat osoittimet
    4. Palomuurin puuttuminen
    5. Natiivi mustin rajoitus 2^32 tavuun
    6. Symmetriset avain algoritmit on helpompi murtaa
    7. TCP porttien määrä on 2^16
    8. Järjestelmän suojaksuun käytetään MD5 ja SHA-1 yhdistelmää

    Omien tietojen ja Googlen mukaan ainoa eri 32 bittisellä (x86) ja 64 bittisellä (x86_64) järjestelmillä on muistin rajoitus 4 Gigatavuun, eli vastaus on 5.

    15. Oikeutta!

    Tiedostolla on oikeudet -rwx-wxr-x. Mitkä seuraavista rajoittavat omistus ryhmän oikeuksia?

    1. chmod 322 file
    2. chmod 333 file
    3. chmod 700 file
    4. chmod 735 file
    5. chmod 754 file
    6. chmod 755 file
    7. chmod a-x file
    8. chmod w-r file
    9. chmod u-r,g-r file

    -rwx-wxr-x on sama kuin 735. Sarjassa toinen numero osoittaa ryhmän oikeuksia, eli oikea lotto numero on: 1, 3, 5, 6, 7.

    16. Sanitaatiota

    Mitkä seuraavista haavoittuvuuksista voidaan estää kokonaan kunnollisella käyttäjän syötteen validoinnilla?

    1. Bruteforcing
    2. CSRF
    3. DDOS
    4. Command injection
    5. SQL injection
    6. XSS

    4, 5 ja 6, kuten kohdassa 14. mainittiin. Brute forcing on salasanojen murtamista (kuten tehtävässä 8. Salasanojen varjostus todettiin). CSRF on sivuston suorittama hyökkäys käyttäjän selaimen kautta. DDOS on palvelunestohyökkäys jolla ei ole tekemistä käyttäjän syötteen kanssa.

    17. Selaimen skriptikielet

    Mitkä seuraavista ohjelmoint/skripti/markup kielistä yleensä pyörivät käyttäjän puolella (esim. web selaimessa)?

    1. ASP
    2. ASP.NET
    3. C#
    4. CSS
    5. HTML
    6. Javascript
    7. PHP
    8. Ruby
    9. SQL

    Oikeat vastaukset: 4, 5, 6. Loput ovat palvelinpään kieliä.

    18. Palomuuri

    Mikä seuraavista on tyypillistä palomuurille?

    1. Estää epäilyttävien ohjelmien ajo
    2. Koodata datasiirto käyttäen UDP protokollaa
    3. Suodattaa TCP paketit jotka eivät vastaa sen sääntöjä
    4. Mahdollistaa SSH verkko protokollan käyttö
    5. Verkko-osoitteiden kääntäminen
    6. Tietokoneen tutkiminen ja käyttäjän varoittaminen jos haittaohjelmia löydetään.

    Palomuurilla on yksi tehtävä. Suodattaa liikennettä, eli ainoastaan 3. kuuluu palomuurin tehtäviin.

    19. Järjestelmän suojaaminen

    Mitkä seuraavista ohjeista auttavat järjestelmän suojaamisessa?

    1. Käytä aina vahvaa salasanaa
    2. Antivirus ohjelma pitää poistaa päältä kun käytetään TCP protokollaa
    3. Antivirus ohjelma pitää poistaa päältä kun käytetään UDP protokollaa
    4. Salasanat tulee tallentaa käyttäen vahvaa, kaksisuuntaista salausta
    5. Järjestelmän tulee automaattisesti kirjata käyttäjän ulos muutaman minuutin seisonnan jälkeen
    6. Uusinta versiota järjestelmästä tulee välttää sillä siinä saattaa olla tuntemattomia turva-aukkoja
    7. Järjestelmän lokit tulee poistaa viikoittain

    Vahvat salasanat ovat hyväksi, mutta ei niitä tule tallentaa kaksisuuntaisella salauksella. Uusi versio järjestelmästä tulee ensiksi testata ettei se häiritse järjestelmää, mutta päivityksiä tehdään syystä ja järjestelmä tulee aina päivittää mahdollisimman pian. Lokit ovat hyödyllisiä tietoturvalle. Ainoa oikea kysymys on käyttäjien uloskirjaus seisonnan jälkeen. Se voi olla hyvä käytäntö jos käyttäjän ymmärtävät sen arvon, mutta todellisuudessa useat käyttäjän tulevat varmaan sen takia käyttämään lyhyitä salasanoja, joten vastaan: 1.

    20. SQL

    Mikä seuraavista SQL lauseista valitsee kaikki rivit pöydästä Customers jossa kolumnin Age arvo on suurempi kuin 19, mutta pienempi kuin 21?

    1. SELECT Customers WHERE Age > 19 and Age < 21
    2. SELECT Customers IF Age > 19 and Age < 21
    3. SELECT * FROM Customers WHERE Age > 19 and Age < 21
    4. SELECT * FROM Customers IF Age > 19 and Age < 21
    5. IF Age > 19 and Age < 21 THEN SELECT Customers

    Vastaus on 3. SELECT * FROM Customers WHERE Age > 19 and Age < 21.

  • PVCC #12: 128 sattumanvaraista bittiä

    27.7.2015

    Etkö tiedä mikä on PVCC? Klikkaa tästä.

    12. tehtävässä meille annetaan 128 merkkinen täysin sattumanvarainen bittijono, josta saamme flipata yhden bitin ympäri. Kysymys kuuluu:

    Kuinka monta bittiä informaatiota voidaan siirtää tällä bitti jonolla?

    Tämä alkaa jo olemaan suhteellisen haastava matemaattinen kysymys, mutta samalla viimeinen “teoria” kysymys.

    Aloitetaan helposta. Tehtävässä tulee käyttää pariteettia. Jos meillä on sattumanvarainen neljän bitin jono, kuten 0110 voidaan sillä koodata kaksi bittiä informaatiota. Pariteetti bitit lasketaan kahden potenssien väliltä, eli \(2^0\) askelväli on 1, \(2^1\) askelväli on 2, mikä on suurin askelväli, mitä neljän bitin jonossa voidaan suorittaa.

    \(2^0\) vastaa loogista AND funktiota 01 kanssa. \(2^1\) vastaa loogista AND funktiota 10 kanssa. Ja niin edelleen.

    Pariteetti saadaan laskemalla kaikki alueen 1, jos niitä on parilline määrä aroksi tulee 0 jos pariton 1.

    Esimerkiksi jonossa 0110 \(2^0\) pariteetti on 1 koska se saa arvon 01. Jonon \(2^1\) pariteetti on myös 1 koska se saa myös arvon 01. Näin ollen loogisten AND funktioiden jälkeen pariteetti jonolle on 11. Jos nyt halutaan osoittaa jotain arvoa mikä voidaan jonolla koodata (koska jono on sattumanvarainen voidaan sillä koodata vain kahden bitin arvo, eli 0-3), esim. numero 2, käytetään vedetään arvo 10 loogisella XOR functiolla lävitse jonon pariteetista: \(10 \oplus 11 = 01\). Näin ollen bitti kohdassa 01 tulee kääntää, eli saamme arvon 0100 (koska ensimmäinen bitti on arvo 00). Nyt kun bittijonoa dekoodataan, eli yritetään saada se arvo ulos jonka lähetimme (numero 2), lasketaan uudesta jonosta pariteetti ja se vedetään lävitse itse jonosta. Jonon 0100 \(2^0\) pariteetti on 0 koska se saa arvon 00. Jonon \(2^1\) pariteetti on 1 koska se saa arvoksi 10. Loogisten AND funktioiden jälkeen saamme arvon 10. Kun käännetään bitti kohdasta 01 saadaan uudeksi jonoksi 0110, kun nyt tämä uusi arvo ja pöydän arvo vedetään lävitse loogisesta XOR funktiosta saadaan selville lähetetty arvo. \(0100 \oplus 0110 = 0010\), eli 2.

    Tällä logiikalla aina kun satunnaisen bittijonon, josta saadaan kääntää yksi bitti mielivaltaisesti, pituus tuplaantuu voidaan sillä siirtää yksi bitti lisää informaatiota, eli kahdeksalla satunnaisella bitillä voidaan siirtää kolme bittiä informaatiota. Näin ollen 128 merkkiä pitkällä sattumanvaraisella bitti jonolla voidaan siirtää seitsemän bittiä informaatiota.

  • PVCC #11: Todennäköisyys

    25.7.2015

    Etkö tiedä mikä on PVCC? Klikkaa tästä.

    11. kysymyksessä on pitkä tekstinpätkä josta puolet voi heti heittää menemään. Oikea kysymys kuuluu:

    “Jos huoneessa on kolme tietokonetta, joilla kaikilla on yhtäläinen mahdollisuus pyörittää Windows, Linux tai OS X käyttöjärjestelmää, mikä on todennäköisyys että huoneessa on vähintään kaksi Linux konetta?”

    Koska vaihtoehtoja on vähän, aloitetaan totuustaulukolla selventämään tilannetta, tässä taulukossa on kaikki tilat jotka täyttävät kysymyksen kriteerit.

    L, L, L       L = Linux
L, L, W       W = Windows
W, L, L       X = OSX
L, W, L
L, L, X
X, L, L
L, X, L

    Kuten huomataan kokoonpanoja, joissa on vähintään kaksi Linux konetta, on \(7\). Kaiken kaikkiaan mahdollisia kokoonpanoja on \(3^3\), eli \(27\), joten todennäköisyys \(P\) on.

    Mikä tarkoittaa että todennäköisyys sille, että huoneessa on vähintään kaksi Linux konetta, on \( \approx 26 \%\).

  • PVCC #10: Regex

    25.7.2015

    Etkö tiedä mikä on PVCC? Klikkaa tästä.

    10. kysymys käsittelee yleisön suosikkia: Regular Expressioneja!

    Meidän tulee tehdä Regex, joka sopii sanoihin jotka alkaa ja loppuu samalla konsonantilla ja joiden pituus on jaollinen kolmella. Regex ei saa olla yli 40 merkkiä pitkä ja jokaisesta merkistä 30 jälkeen sakotetaan.

    Ensimmäinen yritys oli aika karkea, kuten yleensä.

    ^([cdhjklmnnprstvbfgz])+.+([cdhjklmnnprstvbfgz])$

    Mikä ei ihan tehnyt mikä piti.

    >>> import regex
>>> strings = ['test', 'teeest', 'asd', 'lol', 'dead', 'cat']
>>> for s in strings:
...   if regex.match(r'^([cdhjklmnnprstvbfgz])+.+([cdhjklmnnprstvbfgz])$', s):
...     print(s)
...
test
teeest
lol
dead
cat

    Lisäksi len('^([cdhjklmnnprstvbfgz])+.+([cdhjklmnnprstvbfgz])$') on 49, eli regex on aivan liian pitkä. Sitten aloin lukemaan POSIX regular expressionin dokumentaatiota ja löysin regexp ryhmät. Ryhmä on regex jok on sulkujen välissä ja niihin voi viitata ryhmän järjestysluvulla esim. \1. Pistetään käytäntöön.

    ^([cdhjklmnnprstvbfgz]).*\1$

    Mikä on jo parempi, kuten seuraavasta voidaan huomata sana cat on kadonnut, eikä pituus ole enään kuin 27 merkkiä.

    >>> import regex
>>> strings = ['test', 'teeest', 'asd', 'lol', 'dead', 'cat']
>>> for s in strings:
...   if regex.match(r'^([cdhjklmnnprstvbfgz]).*\1$', s):
...     print(s)
...
test
teeest
lol
dead

    Tässä kohtaa jäin vähän jumiin. Koetin erilaisia juttuja (){3} tyylisellä syntaksilla, muttei mikään ottanut tuulta alleen. Onneksi olin heittänyt tämän pallon jo alkuvaiheella myös kaverilleni joka keksi seuraavan parannuksen.

    ^([dhjklmnnprstvbfgz]).(...)*\1$

    Ja se toimii täydellisesti.

    >>> import regex
>>> strings = ['test', 'teeest', 'asd', 'lol', 'dead', 'cat']
>>> for s in strings:
...   if regex.match(r'^([dhjklmnnprstvbfgz]).(...)*\1$', s):
...     print(s)
...
teeest
lol

    Mutta meillä on nyt pienehkö pulma. Pituus regexillä on 31 merkkiä, mikä on toki alta 40 merkkiä, mutta juuri yli 30 merkin. Sitten saimme sellaisen “Ahaa!” ajatuksen, koska regexiä ajetaan kasaa suomenkielisiä sanoa vastaan meidän ei kannata etsiä konsonantteja vaan sulkea pois vokaaleja ja saamme seuraavan.

    ^([^aeiouy]).(...)*\1$

    Tämä regex toimii aivan kuten edellinen, mutta on vain 21 merkkiä pitkä, mikä on reippaasti tehtävän 30 merkki rajan alapuolella.
