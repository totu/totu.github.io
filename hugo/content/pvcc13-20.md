Koska tehtävät 13. - 20. ovat lähinnä Googlen käyttöä pistän ne kaikki tähän yhdeksi postauksesta. Käyn tehtävät lävitse hyvin pintapuolisesti, jos jotain kiinnostaa tietää tarkemmin niin Wikipedia on aika hyvä lähde näille.

# 13\. Selaimella hyökkäys

>Mitä seuraavista haavoittuvuuksia voidaan hyödyntää käyttämällä vain normaalia web selainta (kuten Mozilla Firefoxia tai Google Chromea)?

1. Directory traversal attack
2. LDAP injection
3. Command injection
4. Poison Null Byte
5. SQL injection
6. XSS

Tässä tehtävässä oletan että "normaaliksi" selaimeksi lasketaan myös helposti saatavat selain lisäosat, ne eivät toki ole pakollisia kaikkiin hyökkäyksiin, mutta helpottavat niitä huomattavasti. Kaikki listatut hyökkäykset ovat käyttäjän syötteen sanitoimatta jättämisestä johtuvia ongelmia, eli kaikkiin vastaus on **kyllä**, kaikkia näitä haavoittuvuuksia voi hyväksikäyttää vain selaimella ilman erillisiä työkaluja.

# 14\. 32 bittiä vs. 64 bittiä

>Mitkä seuraavista ovat tyypillisiä x86 järjestelmälle, mutta ei x86_64 järjestelmälle?

1. Puskurin ylivuoto haavoittuvuus
2. DDOS hyökkäyksiä on usein vaikea estää
3. Roikkuvat osoittimet
4. Palomuurin puuttuminen
5. Natiivi mustin rajoitus 2^32 tavuun
6. Symmetriset avain algoritmit on helpompi murtaa
7. TCP porttien määrä on 2^16
8. Järjestelmän suojaksuun käytetään MD5 ja SHA-1 yhdistelmää

Omien tietojen ja Googlen mukaan ainoa eri 32 bittisellä (x86) ja 64 bittisellä (x86_64) järjestelmillä on muistin rajoitus 4 Gigatavuun, eli vastaus on 5.

# 15\. Oikeutta!

>Tiedostolla on oikeudet -rwx-wxr-x. Mitkä seuraavista rajoittavat omistus ryhmän oikeuksia?

1. chmod 322 file
2. chmod 333 file
3. chmod 700 file
4. chmod 735 file
5. chmod 754 file
6. chmod 755 file
7. chmod a-x file
8. chmod w-r file
9. chmod u-r,g-r file

-rwx-wxr-x on sama kuin 735. Sarjassa toinen numero osoittaa ryhmän oikeuksia, eli oikea lotto numero on: 1, 3, 5, 6, 7.

# 16\. Sanitaatiota

> Mitkä seuraavista haavoittuvuuksista voidaan estää kokonaan kunnollisella käyttäjän syötteen validoinnilla?

1. Bruteforcing
2. CSRF
3. DDOS
4. Command injection
5. SQL injection
6. XSS

4, 5 ja 6, kuten kohdassa 14. mainittiin. Brute forcing on salasanojen murtamista (kuten tehtävässä [8. Salasanojen varjostus](/2015/07/23/pvcc8.html) todettiin). CSRF on sivuston suorittama hyökkäys käyttäjän selaimen kautta. DDOS on palvelunestohyökkäys jolla ei ole tekemistä käyttäjän syötteen kanssa.

# 17\. Selaimen skriptikielet

> Mitkä seuraavista ohjelmoint/skripti/markup kielistä yleensä pyörivät käyttäjän puolella (esim. web selaimessa)?

1. ASP
2. ASP.NET
3. C#
4. CSS
5. HTML
6. Javascript
7. PHP
8. Ruby
9. SQL

Oikeat vastaukset: 4, 5, 6. Loput ovat palvelinpään kieliä.

# 18\. Palomuuri

>Mikä seuraavista on tyypillistä palomuurille?

1. Estää epäilyttävien ohjelmien ajo
2. Koodata datasiirto käyttäen UDP protokollaa
3. Suodattaa TCP paketit jotka eivät vastaa sen sääntöjä
4. Mahdollistaa SSH verkko protokollan käyttö
5. Verkko-osoitteiden kääntäminen
6. Tietokoneen tutkiminen ja käyttäjän varoittaminen jos haittaohjelmia löydetään.

Palomuurilla on yksi tehtävä. Suodattaa liikennettä, eli ainoastaan 3. kuuluu palomuurin tehtäviin.

# 19\. Järjestelmän suojaaminen

>Mitkä seuraavista ohjeista auttavat järjestelmän suojaamisessa?

1. Käytä aina vahvaa salasanaa
2. Antivirus ohjelma pitää poistaa päältä kun käytetään TCP protokollaa
3. Antivirus ohjelma pitää poistaa päältä kun käytetään UDP protokollaa
4. Salasanat tulee tallentaa käyttäen vahvaa, kaksisuuntaista salausta
5. Järjestelmän tulee automaattisesti kirjata käyttäjän ulos muutaman minuutin seisonnan jälkeen
6. Uusinta versiota järjestelmästä tulee välttää sillä siinä saattaa olla tuntemattomia turva-aukkoja
7. Järjestelmän lokit tulee poistaa viikoittain

Vahvat salasanat ovat hyväksi, mutta ei niitä tule tallentaa kaksisuuntaisella salauksella. Uusi versio järjestelmästä tulee ensiksi testata ettei se häiritse järjestelmää, mutta päivityksiä tehdään syystä ja järjestelmä tulee aina päivittää mahdollisimman pian. Lokit ovat hyödyllisiä tietoturvalle. Ainoa oikea kysymys on käyttäjien uloskirjaus seisonnan jälkeen. Se voi olla hyvä käytäntö jos käyttäjän ymmärtävät sen arvon, mutta todellisuudessa useat käyttäjän tulevat varmaan sen takia käyttämään lyhyitä salasanoja, joten vastaan: 1.

# 20\. SQL

>Mikä seuraavista SQL lauseista valitsee kaikki rivit pöydästä Customers jossa kolumnin Age arvo on suurempi kuin 19, mutta pienempi kuin 21?

1. SELECT Customers WHERE Age > 19 and Age < 21
2. SELECT Customers IF Age > 19 and Age < 21
3. SELECT * FROM Customers WHERE Age > 19 and Age < 21
4. SELECT * FROM Customers IF Age > 19 and Age < 21
5. IF Age > 19 and Age < 21 THEN SELECT Customers

Vastaus on 3. `SELECT * FROM Customers WHERE Age > 19 and Age < 21`.
